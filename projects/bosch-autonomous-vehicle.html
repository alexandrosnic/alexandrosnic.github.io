Optimization of Sensory Data Exchange

Squeezing microsecond latency out of a production-grade autonomous vehicle system

The Challenge

In high-level autonomous driving (Level 4), the bottleneck is often not just the AI algorithm, but the middleware's ability to transport massive sensor data (Lidar, Radar, Cameras) to the CPU/GPU with zero packet loss and minimal latency. Working within Bosch’s Autonomous Driving division, I was tasked with optimizing a stack that was already considered highly efficient.

My Contribution & Impact

I moved beyond standard application-level development to perform low-level systems engineering. My work focused on identifying micro-bottlenecks that classical profilers often miss. By diving into the Linux Kernel network stack and utilizing tools like Wireshark and eBPF, I pinpointed latency spikes in standard UDP/ETH communication.

I researched and implemented RDMA (Remote Direct Memory Access) and shared memory transport protocols, successfully increasing sensory data throughput and reducing latency by 10%. This optimization ensured that the perception layer received high-frequency data under strict real-time constraints, directly improving the vehicle's reaction time.

Technical Deep Dive

  • Protocol Optimization: Benchmarked and implemented RDMA (SMC-R) to bypass kernel overhead for high-speed data transfer.
  • Kernel Debugging: Analyzed network interface bottlenecks and tuned kernel parameters for optimized interrupt handling.
  • Hybrid Middleware: Developed robust containerized environments (Docker) supporting both ROS 1 and ROS 2 communication for local and embedded targets.
  • High-Performance Profiling: Utilized Intel VTune and perf to optimize C++/CUDA code at the instruction level.
