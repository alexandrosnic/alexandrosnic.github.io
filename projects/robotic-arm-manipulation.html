Unified ROS2 Manipulation Framework

Architecting a User-Friendly Hub for Human-Robot Collaboration (COCOBOTS)

The Challenge

In the COCOBOTS project, the goal was to enable human-robot collaboration in industrial settings. This required a complex system where Natural Language Processing (NLP) for commands, Computer Vision (CV) for object detection, and Motion Planning for execution could all talk to each other. The challenge was the "Research Drag": integrating disparate research modules into a stable, real-time framework capable of running on physical industrial hardware (UR3e).

My Contribution: The System Architect

I architected a unified, Dockerized ROS2 framework that served as the central nervous system for the project. By creating a robust API, I enabled the NLP and CV teams to plug their models directly into the robotic control pipeline without needing deep expertise in robotics middleware.

I pioneered the Sim-to-Real pipeline using Webots and MoveIt2. This allowed for high-fidelity testing in simulation that translated seamlessly to the physical UR3e arm. To bridge hardware gaps, I migrated legacy ROS1 drivers for the OnRobot VGC10 vacuum gripper to ROS2 and designed custom mechanical components in SolidWorks to optimize the gripper's performance for specific industrial pick-and-place tasks.

Key Engineering Achievements

  • ROS2 Migration: Led the transition from ROS1 to ROS2 Foxy, ensuring improved real-time performance and system stability.
  • State Machine Logic: Implemented complex task logic using SMACH to handle error recovery and sequential pick-and-place workflows.
  • Hardware-In-The-Loop: Established a seamless bridge between Webots simulations and physical UR3e hardware via custom proto/URDF files.
  • Mechanical Design: Engineered and 3D-printed custom adapters and vacuum suction arrays in SolidWorks to handle non-uniform industrial objects.
