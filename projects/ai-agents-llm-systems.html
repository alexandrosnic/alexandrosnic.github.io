LLM-powered AI Agents for Robotic systems

Lead developer of a robotic startup's AI agent stack

Description

I designed and implemented the AI agent project as an LLM-agent orchestration layer for engineering and robotics workflows at Conbotics. The system was built to move beyond single-shot prompting into structured, multi-step execution with tool use, context management, and deterministic task decomposition.

Based on the local implementation, I built a fully local RAG assistant stack with Ollama-hosted models, ChromaDB persistence, hybrid retrieval (BM25 + vector similarity), and Streamlit UX for technician-facing troubleshooting. To keep sensitive infrastructure protected, this page describes architecture only at a high level and excludes private operational content.

Capabilities Demonstrated

  • Agent Architecture: Built modular planning/execution loops for reliable multi-step task completion.
  • Context and Retrieval: Implemented structured context injection, source citations, and domain-specific troubleshooting retrieval.
  • Quality and Safety: Added strict grounding, unknown-admission behavior, and input-hardening for safer operational use.

Technologies Used

  • Ollama-hosted LLMs (local)
  • Agent orchestration patterns
  • LangChain / LlamaIndex / smolagents
  • ChromaDB + hybrid BM25/vector retrieval
  • Streamlit
  • Prompt engineering and evaluation loops
  • Python
